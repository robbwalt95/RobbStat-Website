Project Details

College course: CS492 Computer Science Team Project Part 2

In a group size of four, we were assigned to automate a Book Store using the Scrum framework.

Project Requirements:

-Keeping track of inventory

-Keeping track of sales records

-Providing the abilities to place orders to manufacturers

-Providing the ability for customers to place orders


Technologies

Communication Method: Discord

IDE & Coding Sessions: IntelliJ

Product Backlog: Google Sheets

Product Doc: Google Docs

Frontend: HTML, CSS, Javascript

Backend: Java, Spring Framework, H2 Database


Personal Implementation and Experience

The results of this project were accomplished though team effort.

This project was two Sprints. With a total of two weeks per Sprint.

Since I was not with this team for part one, I had a bit to catch up on. Unfortunately, the team members did not coordinate much for the first half. Luckily, this gave me the opportunity to gather, edit, and ultimately connect tasks/stories with each member for the Backlog. For development, we decided to split into two groups. One to focus on the frontend and the other the backend. I was selected for the backend. The roles of developer, Product Owner, and Scrum Master were rotated between each member.

The primary tasks I accomplished as a Backend Dev include:

-Initializing Spring Boot and the H2 (In memory) database

-Feeding data into the database

-Implementing the ability to query data as an administrator

-Connecting Spring and Web UI with Hibernate


The primary tasks I accomplished as Product Owner include:

-Adjusted tasks per Sprint to accommodate deadlines/Stake Holder expectations

-Used the Project Requirements as reinforcement for project direction


The primary tasks I accomplished as Scrum Master include:

-Hosted one on one, outside group meetings, and Retros to understand member impediments

-Insisted on Coding Sessions where critical feedback was professionally handled on all sides
